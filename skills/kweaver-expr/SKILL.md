# KWeaver Expr

管理团队 KWeaver 平台经验库。经验库存放在用户本地 `~/.expr/kweaver/` 目录（**不在本仓库**），由本技能负责读写。

## 经验库结构

```
~/.expr/kweaver/
├── INDEX.md             # 一行摘要索引——load 时先读这个
├── concepts.md          # 平台概念 / 配置 schema
├── usage.md             # API 调用 / CLI 命令 / 响应解析
├── troubleshooting.md   # 具体报错与解法
└── gotchas.md           # CLI 行为坑 / 平台限制 / 待修工单
```

每条经验**新增**时使用以下结构（已存在的旧条目不强制改格式，只在下次 update 时顺手转换）：

```markdown
### {简短标题}

**场景:** 何时会遇到 / 触发条件
**做法/坑:** 怎么做 / 注意什么 / 错误做法
**为什么:** 根因 / 设计原因（可选但强烈推荐）
**最后验证:** YYYY-MM-DD（可选标签：KWeaver vX.Y / 环境）
```

## 模式分发

通过 args 参数区分：
- 无 args 或 `load` → load 模式
- `save` → save 模式
- `prune` → prune 模式
- 其他文本 → 当作话题关键字走 load 模式（用关键字精筛 INDEX）

## load 模式（默认）

1. Read `~/.expr/kweaver/INDEX.md`。如果不存在，告知用户经验库尚未初始化，建议先 `save` 或手动建库后退出。
2. 根据当前对话上下文（用户最近提到的关键词、报错信息、操作意图），从 INDEX 中筛出最多 **5 条**最相关条目。
3. 对每条相关条目，Read 对应分类文件的对应小节，把内容简要呈现给用户：
   - 标题
   - 场景 / 做法 / 为什么 三段（如有）
   - 最后验证日期（用于判断时效性）
4. 如果命中的条目「最后验证」早于 90 天前，在呈现时附一行 `⚠️ 该条目超过 90 天未验证，可能已过期`。
5. 不要把整个分类文件全文塞回——只输出命中的小节。

## save 模式

1. Read 完整 `INDEX.md` 和 4 个分类文件。
2. 从当前对话中提炼可沉淀经验。**只沉淀满足以下三条的内容**：
   - **可复用**：未来类似场景还会遇到
   - **非显然**：不是查文档/读代码就能立刻知道的
   - **已验证**：本次对话中确实跑通或确认过
3. 拒绝沉淀的内容：
   - 「这次我做了 X」式的日记/活动记录
   - 一次性配置 / 临时调试痕迹
   - 文档里能直接查到的标准用法
4. 对每条候选经验，**先在已有条目中找最近邻**：
   - 标题或场景明显重叠 → 走 **update**：合并新观察、刷新「最后验证」、保留历史 nuance
   - 完全新主题 → 走 **append**：选择最匹配的分类文件追加；只在 4 类都不合适时新建分类文件并更新 SKILL.md 提及（罕见）
5. 同步更新 `INDEX.md`：每条一行，格式 `- [{文件}#{标题}](./{文件}.md) — {一句话摘要}`，按主题分组。
6. 把当前日期（`date +%Y-%m-%d`）填入「最后验证」字段。
7. 如果同一主题下已积累 > 8 条且明显有重叠，提示用户考虑合并整理（不要自动合并）。
8. 报告变更摘要：新增 N 条 / 更新 M 条 / 跳过 K 条（每条跳过附原因）。

## prune 模式

目的：标记可疑过期条目让用户人工确认，**不直接删除**。

1. Read 4 个分类文件，挑出「最后验证」早于 90 天前的条目（无日期的旧条目也算）。
2. 对每条，结合当前对话或可执行的轻量验证（grep 仓库 / `kweaver --help` / swagger 等）判断状态：
   - **仍成立** → 把「最后验证」刷新为今天日期
   - **明确已变** → 在条目首行追加 `<!-- stale since YYYY-MM-DD: {简要理由} -->`，**不删除**正文
   - **无法判断** → 加 `<!-- stale? since YYYY-MM-DD -->`
3. INDEX.md 中给标记为 `stale` / `stale?` 的条目摘要前加 `⚠️ ` 前缀。
4. **绝不**自动删除任何条目——删除是不可逆操作，需用户在另一次明确请求中执行。
5. 报告：刷新 N 条 / 标记 stale M 条 / 标记 stale? K 条。

## 触发约定

以下用户输入应被视为本技能的触发信号：
- 提到 KWeaver / kweaver / agent-factory / agent-executor / dolphin / SKILL 注册 / kweaver CLI
- 描述 agent 配置 / 编排 / chat completion 响应解析问题
- 末尾出现"记一下" / "沉淀一下" / "存到经验里" → 视为 `save` 意图
- "查一下 KWeaver 经验" / "之前怎么解决的" → 视为 `load` 意图

## 关键纪律

- **写入前必须先 Read 现有内容**——避免覆盖、避免重复条目。
- **不擅自跨分类移动条目**——除非用户明确要求重整。
- **不删除任何内容**——prune 只标记，删除由用户显式请求触发。
- **保留原始 prose**——迁移/合并时不要为了"工整"而牺牲原有的具体细节、错误信息原文、命令片段。
